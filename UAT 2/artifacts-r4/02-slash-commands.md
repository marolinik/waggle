## Root Cause Analysis

The core issue is architectural: most slash commands delegate to `ctx.runWorkflow()` or `ctx.getWorkspaceState()`, and when these context methods are unavailable or return empty, there is no graceful degradation. The commands were designed for a fully-wired agent context but are being invoked via the HTTP chat endpoint where the workflow runner is not connected.

**Three categories of failure**:

1. **No fallback at all** (`/research`, `/plan`): Return raw "not available" error. These should degrade to LLM-only responses.
2. **Empty fallback** (`/catchup`, `/status`): Return "no state available." These should search memory for relevant frames.
3. **Template-only output** (`/draft`, `/decide`): Return scaffolding without content. These should invoke the LLM to fill in the template.

**Recommendation**: Every slash command should have a meaningful LLM-powered fallback that produces useful output even when the workflow runner and workspace state are unavailable. The command system should never return a raw developer error message to the user.
